# concepts

「これまでの研究により明らかになっている概念」を蓄積する場所。
新しいinsightを生成する際の照合先となり、差分から研究のヒントが生まれる。

## 役割

- 研究で「すでにわかっていること」の集積地
- 新しいソースと突き合わせることで、支持・修正・対立・拡張を判定する基盤
- 各conceptに含まれる「研究のヒント」がRQ生成の種になる

## 各ファイルに書くべきこと

- 概念の定義・記述
- その概念が浮上した背景（どのsource/議論から生まれたか）
- 既存知見との関係（他のconceptsとの差分）
- **研究のヒント**（この概念から示唆される次の問い）
- 論文での活用可能性
