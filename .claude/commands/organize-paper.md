# Organize Paper

指定されたファイル（論文・修士論文等）を読み込み、03_papers/ のディレクトリ構造に整理する。

## 使用方法

引数: `$ARGUMENTS`

**形式**: `<ファイルパス> [オプション...]`

- **ファイルパス**（必須）: 整理したい論文ファイルのパス（01_insight-sources/ 配下の相対パス、または絶対パス）
- ファイルが大きい場合は自動的にチャンクで読み込む

### オプション

| オプション | 説明 | デフォルト |
|---|---|---|
| `--slug <名前>` | ディレクトリ名（例: `tonegawa-2023-chat`） | 自動生成（著者-年-キーワード） |
| `--status <状態>` | `drafting / submitted / in_review / revision / accepted / published` | `accepted` |

### 使用例

```
/organize-paper previous-papers/2023master.txt
```

```
/organize-paper literature/smith-2024.pdf --slug smith-2024-advocacy --status published
```

## 実行手順

### 1. ファイルの読み込み

指定ファイルを読み込む。大きいファイルはチャンク（200行ずつ）で読み、全体の構造を把握する。

**抽出すべき情報:**
- 論文タイトル（日本語・英語）
- 著者・所属
- 年度・日付
- 研究倫理承認番号
- 論文の章立て構造

### 2. ディレクトリの作成

`03_papers/{slug}/` に以下を作成:

```
03_papers/{slug}/
├── _meta.md
├── 01_introduction.md
├── 02_literature-review.md
├── 03_methodology.md
├── 04_findings.md
├── 05_discussion.md
├── 06_conclusion.md
├── citations.md
└── data/
    ├── raw/.gitkeep
    └── processed/.gitkeep
```

### 3. 各セクションへの内容マッピング

原文の構造を以下のセクションにマッピングする。完全な転写ではなく、**構造化された要約**を作成する。

| セクション | マッピング元 | 記載内容 |
|---|---|---|
| `_meta.md` | 表紙・奥付等 | タイトル、著者、日付、ステータス、原文ソースパス |
| `01_introduction.md` | 背景・目的・RQ | 研究背景の要約、研究目的、リサーチクエスチョン |
| `02_literature-review.md` | 先行研究・理論的枠組み | 先行研究の構造表、理論的枠組みの要約 |
| `03_methodology.md` | 研究方法 | デザイン、参加者、データ収集・分析、倫理 |
| `04_findings.md` | 結果 | カテゴリ・テーマの構造表、主要な知見 |
| `05_discussion.md` | 考察 | 主要な考察ポイント、先行研究との比較、提言 |
| `06_conclusion.md` | 結論・限界・展望 | 要約、限界、主要な貢献、今後の研究 |
| `citations.md` | 引用文献 | 全引用文献のリスト（書誌・DOI・引用箇所・引用目的） |

### 4. 記載のルール

- **各セクション冒頭に原文参照先を `<!-- -->` コメントで記載する**（行番号付き）
- 逐語録や詳細データは原文参照とし、セクションファイルには**テーマ・サブテーマの構造表**を中心に整理する
- テーブル・箇条書きを活用し、可読性を優先する
- `_meta.md` に `## 原文ソース` として元ファイルへのパスを必ず記録する

### 5. 完了報告

以下を出力する:
- 作成したディレクトリパス
- 各セクションの概要（1行ずつ）
- 原文ファイルからの構造マッピングの対応表
